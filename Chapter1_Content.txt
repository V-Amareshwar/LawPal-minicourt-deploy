**1.1 Purpose of the Project**

The primary purpose of the LawPal-Minicourt project is to democratize access to basic legal guidance through a scalable, AI-driven assistant. By leveraging modern advancements in Natural Language Processing (NLP) and Retrieval-Augmented Generation (RAG), the system is designed to provide immediate, contextually accurate answers to legal queries. It aims to empower everyday users, students, and legal professionals by automating the parsing of complex legal documents, evaluating case parameters, and acting as a preliminary exploratory tool for legal precedents, such as standard bail calculations, without substituting formal legal counsel. 

**1.2 Problem with Existing Systems**

Existing technical solutions in the legal domain suffer from a variety of significant limitations that restrict their utility for the average user:
*   **High Costs and Inaccessibility:** Most current legal consulting requires human intervention, which is accompanied by high hourly rates and lengthy waiting times.
*   **Complex Legal Jargon:** Traditional legal databases and search engines rely heavily on dense legal terminology, meaning laypersons often struggle to find or understand the precedents relevant to their situations.
*   **Rigid Keyword Searching:** Standard systems often rely on exact keyword matches rather than semantic meaning. This leads to missed information if a user queries "stealing" instead of the strictly defined legal term "larceny" or "theft."
*   **Lack of Contextual Understanding:** Current chatbots that aren't domain-specific tend to hallucinate or provide generic, legally unsound advice because they lack a localized, referenced factual database.

**1.3 Proposed System**

To overcome these barriers, the proposed system is a full-stack, domain-specific AI web application named LawPal-Minicourt. The solution integrates a responsive React-based frontend tailored for seamless user interaction with a robust dual-backend infrastructure. User management, session history, and real-time WebSocket communications are handled by a Node.js/Express server. The core intelligence is driven by a Python (Flask) inference engine that utilizes a custom Retrieval-Augmented Generation (RAG) pipeline. By employing ChromaDB for vector similarity searches alongside BM25 rankings, the system can semantically match a user's natural language query against a pre-processed corpus of legal data before handing it over to a Groq-powered LLM for a formulated, accurate response.

**1.4 Scope of the Project**

The scope of LawPal-Minicourt encompasses the development of an end-to-end web platform functioning primarily as an automated legal Q&A chatbot. Specific deliverables within this scope include:
*   Developing an intuitive User Interface for account management, dashboard viewing, and real-time chat interactions.
*   Creating a secure authentication and session management layer.
*   Implementing a highly optimized RAG pipeline capable of ingesting, chunking, and embedding raw legal data files into a vector store.
*   Ensuring responses generated by the LLM are strictly bounded by the referenced legal context provided by the vector database to prevent hallucinations.
*   Developing specialized modular features, such as an algorithmic bail estimation calculator based on input parameters.
*   *Out of scope:* Providing legally binding representation or acting as a replacement for human lawyers in a court of law.

**1.5 Architecture Diagram**

The system follows a modern, decoupled microservices architecture designed for scalability and low latency. 

*   **Presentation Layer (Client):** A web browser rendering a React.js application, passing data seamlessly using HTTP REST APIs and WebSockets.
*   **API Gateway & Auth Layer:** A Node.js application running Express. It handles user authentication (JWT/bcrypt), rate limiting, routing chat requests, and saving chat history into a MongoDB database.
*   **AI Inference Layer:** A Python server running Flask. When a query is received, it vectorizes the text using `sentence-transformers`, retrieves matching legal segments from the local ChromaDB, and performs a keyword search using BM25.
*   **Generation Layer:** The retrieved contextual segments and the original user query are fused into an enhanced prompt and sent via API to the Groq LLM, which streams back the final reasoning and formulated response to the user.
